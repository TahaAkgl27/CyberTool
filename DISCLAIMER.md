**Last updated:** July 2026  
**Applies to:** All versions of the open source CyberTool software

## Plain-Language Summary

CyberTool is **legitimate security and diagnostics software**. It is **not malware**, **not spyware**, and **not designed for unauthorized access** to computer systems.

You may use CyberTool only in situations where you have **legal authority and explicit permission** to assess the target systems.

---

## What CyberTool Is

CyberTool is an open source Windows application that provides:

- Network port scanning and risk assessment
- Windows system enumeration (including WMI)
- Security posture helpers and reporting
- Optional AI-assisted analysis (user-configured)
- Training-oriented simulations for authorized education

It is intended for **defensive diagnostics, IT operations, and security education**.

## Authorized Use Only

CyberTool should **only** be used in:

### Laboratories

- Isolated virtual machines
- Classroom networks with instructor oversight
- Certification training environments
- Personal learning setups on owned hardware

### Authorized Corporate Environments

- Systems you administer
- Penetration tests with signed rules of engagement
- Internal staging/pre-production with IT approval
- SOC validation with defined scope

### Education

- University and vocational cybersecurity courses
- Security awareness workshops
- Blue team training exercises

### Research

- Academic security research with ethics approval
- Tool evaluation in controlled settings
- Open source contribution and development

---

## Prohibited Use

You must **not** use CyberTool to:

- Access systems without authorization
- Scan third-party networks, ISPs, or public infrastructure without permission
- Bypass authentication on systems you do not own or administer
- Deploy ransomware or destructive payloads (CyberTool simulates awareness—it does not deploy ransomware)
- Violate applicable computer crime, privacy, or telecommunications laws
- Harass, surveil, or harm individuals or organizations

CyberTool includes modules that may:
